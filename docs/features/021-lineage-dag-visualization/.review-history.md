## Design Review - Iteration 1 - 2026-03-08T17:45:00+08:00

**Reviewer:** design-reviewer (skeptic)
**Decision:** Needs Revision

**Issues:**
- [blocker] [consistency] Click syntax uses bare `click nodeId "/url"` — should use `href` keyword for explicit URL link syntax
  Suggestion: Use `click {id} href "/entities/{tid}"` for clarity and future-proofing
- [blocker] [completeness] Click URL uses raw type_id without URL encoding — colons in type_ids could cause issues
  Suggestion: Colons are valid in URL paths; FastAPI `{identifier:path}` handles them; Mermaid treats quoted string as opaque URL. Documented.
- [warning] [completeness] Flat children list shows all descendants with depth=10 — no hierarchy nesting in fallback
  Suggestion: Document as accepted trade-off
- [warning] [feasibility] ESM module load timing — startOnLoad may miss elements if DOMContentLoaded already fired
  Suggestion: Mermaid.run() is called internally after initialize(); documented with fallback
- [warning] [assumptions] Empty type_id edge case for _sanitize_id not documented
  Suggestion: Document edge cases (empty, all-special-chars)
- [warning] [completeness] Dict merge order doesn't guarantee entity dict wins on duplicates
  Suggestion: Reorder to ancestors + children + [entity] so entity is last
- [suggestion] [completeness] Integration test seeding sets parent_type_id but not parent_uuid — get_lineage uses parent_uuid CTE
  Suggestion: Fix _seed_entity_with_parent to set both parent_type_id and parent_uuid

**Changes Made:**
- Added `href` keyword to click handler syntax in both design and spec
- Documented URL encoding rationale (colons safe in paths, Mermaid treats as opaque)
- Added flat list trade-off and ESM timing to risks table
- Documented _sanitize_id edge cases
- Reordered dict merge to `ancestors + children + [entity]` so entity wins
- Fixed _seed_entity_with_parent to set parent_uuid via SQL lookup
